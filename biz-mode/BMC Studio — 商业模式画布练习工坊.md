# BMC Studio — 商业模式画布练习工坊

## 设计概述

BMC Studio 是一个交互式商业模式画布（Business Model Canvas）练习工具，帮助用户通过结构化填写、引导式提问和多维评分来系统练习商业模式设计能力。

### 设计理念

商业模式画布的本质是一个系统的”状态空间描述”——9个模块对应9个状态变量，模块之间的耦合关系才是真正的设计难点。工具的核心目标不是”填表”，而是通过引导性提问和反馈评分，训练用户的因果推理和系统思维能力。

借鉴控制论中的闭环反馈思想：填写 → 评分 → 改进建议 → 修正 → 再评分，形成持续优化的学习回路。

## 功能架构

```
BMC Studio
├── 画布模式（Canvas View）
│   ├── 项目命名
│   ├── 9 模块卡片列表
│   ├── 填写进度条
│   └── 实时评分入口
├── 编辑模式（Edit View）
│   ├── 模块引导提示（Hint）
│   ├── 自由文本输入区
│   └── 自检问题列表（3 题/模块）
├── 评分模式（Score View）
│   ├── 综合等级环形图（S/A/B/C/D）
│   ├── 四维分项评分
│   ├── 针对性改进建议
│   └── 导出 HTML 报告
└── 导出功能
    └── 完整 HTML 报告（含评分 + 全部模块内容）
```

## 九大模块设计

每个模块包含四个层次的信息：

|模块    |英文名               |引导提示（Hint）    |自检问题数|
|------|------------------|--------------|-----|
|🤝 关键合作|Key Partners      |关键供应商和合作伙伴识别  |3    |
|⚙️ 关键活动|Key Activities    |核心业务活动类型判断    |3    |
|💎 核心资源|Key Resources     |资源类型与稀缺性分析    |3    |
|✦ 价值主张|Value Proposition |痛点、差异化、付费意愿   |3    |
|💬 客户关系|Customer Relations|关系类型与升级路径     |3    |
|📡 渠道通路|Channels          |触达效率与成本结构     |3    |
|👥 客户细分|Customer Segments |画像、维度、优先级     |3    |
|📊 成本结构|Cost Structure    |固定/可变成本与规模效应  |3    |
|💰 收入来源|Revenue Streams   |收入模式、定价策略、可持续性|3    |

### 自检问题示例（价值主张模块）

1. 核心痛点是什么？你如何解决？
2. 与竞品的差异化在哪里？
3. 客户愿意为哪个具体价值买单？

## 评分体系设计

### 四维评分模型

|维度                  |权重 |评估逻辑                                                                  |
|--------------------|---|----------------------------------------------------------------------|
|**完整度** Completeness|30%|已填写模块数 / 9，覆盖面是基本功                                                    |
|**深度** Depth        |30%|按内容长度梯度打分（<20字→30分, <60字→55分, <120字→75分, <200字→90分, ≥200字→100分），取均值   |
|**一致性** Coherence   |25%|检测关键模块间的逻辑连接——价值主张↔客户细分、价值主张↔收入来源、客户细分↔渠道通路是否同时有实质内容                  |
|**洞察力** Insight     |15%|扫描全文是否包含深层思考关键词：壁垒、飞轮、网络效应、锁定、护城河、颠覆、痛点、场景、假设、验证、迭代、反馈、闭环、杠杆、稀缺、边际、复用等|

### 等级标准

|等级   |分数区间 |评价       |
|-----|-----|---------|
|**S**|≥ 90 |商业直觉敏锐   |
|**A**|75–89|框架扎实，有深度 |
|**B**|60–74|基础完整，可深挖 |
|**C**|40–59|骨架初现，继续补充|
|**D**|< 40 |刚刚起步，大胆填写|

### 综合评分公式

```
Total = Completeness × 0.30 + Depth × 0.30 + Coherence × 0.25 + Insight × 0.15
```

## 改进建议引擎

根据各维度得分动态生成建议：

|触发条件    |建议内容                                    |
|--------|----------------------------------------|
|完整度 < 80|还有模块未填写，试着把9个模块都覆盖一遍，即使只写几个关键词也好过空白     |
|深度 < 60 |部分模块内容偏少，试着针对每个引导问题展开回答，至少写2-3个要点       |
|一致性 < 70|检查价值主张与客户细分是否对齐——你解决的问题真的是目标客户最痛的点吗？    |
|洞察力 < 60|试着加入更深层的思考：护城河在哪？飞轮效应如何形成？竞争者进入后你的壁垒是什么？|
|全部达标    |框架已经很扎实了！下一步可以做竞品对比画布，或者模拟投资人的质疑来压力测试   |

## 导出功能

生成独立的 HTML 文件，包含：

- 项目名称与导出时间
- 综合等级与四维分项分数
- 全部 9 个模块的填写内容
- 暗色主题，可在浏览器中打开或打印为 PDF

## 视觉设计

### 设计风格

- **基调**：深色编辑器风（#141414 底色），金色（#d4af37）作为主强调色
- **字体**：Georgia + Noto Serif SC 衬线组合，传达专业感与可读性
- **交互**：卡片式模块列表，点击进入全屏编辑，评分页用环形进度图呈现

### 色彩体系

|用途    |色值       |
|------|---------|
|背景    |`#141414`|
|卡片面   |`#252525`|
|主强调（金）|`#d4af37`|
|正文    |`#e8e4df`|
|次要文字  |`#888888`|
|分隔线   |`#333333`|

### 状态反馈

- 已填写模块：左侧金色边框 + ✓ 标记
- 未填写模块：左侧灰色边框 + “点击填写…” 提示
- 进度条：金色渐变填充，实时反映完成比例

## 使用流程

```
1. 命名项目 → 如"面向中小银行的FpML合规SaaS"
2. 逐个点击模块 → 阅读引导提示 → 自由填写 → 参考自检问题
3. 返回画布 → 观察进度和实时评分
4. 进入评分页 → 查看四维分析和改进建议
5. 导出 HTML 报告 → 存档或分享
```

## 技术实现

- **框架**：React（JSX 单文件组件）
- **状态管理**：useState 管理画布数据、视图切换、项目名称
- **评分引擎**：纯前端计算，无后端依赖
- **导出**：Blob API 生成 HTML 文件并触发浏览器下载
- **响应式**：移动端优先设计，卡片列表布局适配小屏

## 可扩展方向

1. **AI 辅助点评**：接入 openai compatible API，对填写内容做更深度的一致性分析和商业逻辑质疑
2. **竞品对比模式**：同时打开两张画布，横向对比自己的设计与竞品
3. **投资人压力测试**：模拟 VC 视角的尖锐提问，检验商业模式的鲁棒性
4. **时间轴推演**：从静态画布扩展到”年度演化”模式，模拟商业模式在不同阶段的形态变化
5. **模板库**：预置经典商业模式模板（SaaS、平台、直销等），支持一键加载后修改
